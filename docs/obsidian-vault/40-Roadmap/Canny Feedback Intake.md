# Canny Feedback Intake

Last reviewed: 2026-03-26

Source:

- https://stop-the-slop.canny.io/feature-requests

## Current State

The public Canny feature request board currently shows no posts.

Observed state on review:

- board: `Feature Requests`
- visible statuses: `Under Review`, `Planned`, `In Progress`, `Complete`
- result state: `There are no posts.`

## What This Means

Right now, Canny is not yet a meaningful source of product signal.

It should be treated as an empty intake surface, not as an active backlog driver.

The current product direction should still be driven by:

- direct hands-on product iteration
- community behavior on the site
- Discord discussion
- research notes in the vault

## Intended Role For Canny

Once it has real usage, Canny should act as the structured product-feedback layer for:

- feature requests
- quality-of-life fixes
- navigation and UX complaints
- moderation and trust requests
- workflow requests from active users

It should not replace the board itself.

The board is for AI experiences.

Canny is for feedback about the StopTheSlop product.

## Integration Rule

When Canny begins receiving real posts:

1. Review the highest-voted and newest items on a regular cadence.
2. Normalize them into a small number of product themes.
3. Reflect those themes in:
   - [[40-Roadmap/Roadmap]]
   - [[40-Roadmap/Open Questions]]
   - relevant implementation notes
4. Avoid copying Canny verbatim into the roadmap.
5. Convert requests into product decisions, open questions, or explicit backlog items.

## Candidate Theme Buckets

Useful future buckets for Canny items:

- posting flow friction
- board browsing and search
- wiki quality and accuracy
- account and identity
- moderation and abuse handling
- Discord integration
- import and crawler workflows
- analytics and admin tooling

## Current Recommendation

Do not overinvest in Canny process yet.

Keep the board open, but only start a formal review cadence once there is enough real traffic to produce repeated requests.
